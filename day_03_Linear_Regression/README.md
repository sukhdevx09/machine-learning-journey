# Day 03 – Linear Regression & Encoding Techniques

Today I implemented Linear Regression on a real dataset from Kaggle and explored encoding techniques.

---

## 📊 Concepts Learned

### 🔹 Linear Regression
- Used for predicting continuous values  
- Finds relationship between features and target variable  

---

### 🔹 Cost Function
- Measures how well the model performs  
- Lower cost = better model  

---

### 🔹 R² Score
- Measures model performance  
- Range: 0 to 1  
- Higher value = better fit  

---

### 🔹 Encoding Techniques

#### 1. One-Hot Encoding
- Converts categorical variables into multiple binary columns  
- Avoids ordinal relationship  

---

#### 2. Label Encoding
- Assigns numeric values to categories  
- Simple but may introduce unintended order  

---

## 🧪 What I Implemented

- Performed EDA and data cleaning  
- Applied Label Encoding  
- Applied One-Hot Encoding  
- Trained Linear Regression model on both datasets  
- Compared model performance using R² score  

---

## 📈 Key Observations

- One-Hot Encoding generally performs better for nominal data  
- Label Encoding is simpler but may reduce accuracy  
- Data preprocessing significantly impacts model performance  

---

## 🔄 Workflow

Dataset → EDA → Data Cleaning → Encoding → Model Training → Evaluation

---

## 🔑 Key Takeaways

- Encoding plays a crucial role in ML models  
- Model performance depends heavily on preprocessing  
- Linear Regression is a good starting model
